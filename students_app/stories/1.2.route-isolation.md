# Story 1.2: Route Isolation & Security Hooks

**Status:** `Draft`
**Epic:** 1. Foundation & Camper Authentication

### User Story

**As a** Security framework,
**I want** to intercept requests to the `/(camper)` route group and manage authentication contexts,
**so that** only authenticated students can access the portal, and students are prevented from accessing the staff application.

### Acceptance Criteria

1.  **Route Group Created:** A new `src/routes/(camper)` directory exists to house the student portal.
2.  **Type Definitions Updated:** `App.Locals` in `src/app.d.ts` includes `studentPb` and `student` (typed as `StudentsResponse`).
3.  **Hooks Implementation:** `src/hooks.server.ts` instantiates a third PocketBase instance using the `pb_student_auth` cookie.
4.  **Student Guard:** Accessing any route under `/(camper)` (except public ones like `/login`) without a valid student session redirects to `/(camper)/login`.
5.  **Role Isolation:** A logged-in Student trying to access `/(app)` routes is redirected to `/(camper)`.
6.  **Staff Access:** Existing Staff/Admin authentication flows remain unaffected. Staff accessing `/(camper)` are allowed (for testing/intervention purposes) or redirected based on final logic (Default: Allow for now).
7.  **Cookie Management:** The `pb_student_auth` cookie is properly exported/set in the response headers alongside existing cookies.

### Tasks / Subtasks

- [ ] **1. Update Global Types (`src/app.d.ts`)**
  - [ ] Import `StudentsResponse` from `$lib/pocketbase-types`.
  - [ ] Add `studentPb: TypedPocketBase` to `App.Locals`.
  - [ ] Add `student: StudentsResponse | null` to `App.Locals`.
- [ ] **2. Scaffold Camper Route Group**
  - [ ] Create directory `src/routes/(camper)`.
  - [ ] Create a basic `src/routes/(camper)/+layout.svelte` (placeholder slot).
  - [ ] Create a basic `src/routes/(camper)/+page.svelte` (placeholder "Cockpit" dashboard).
  - [ ] Create a basic `src/routes/(camper)/login/+page.svelte` (placeholder login page to serve as redirect target).
- [ ] **3. Implement Multi-Tenant Hooks (`src/hooks.server.ts`)**
  - [ ] **Instantiation:** Create `studentPb` using `createPocketBase('pb_student_auth')`.
  - [ ] **Load:** Load auth store from request cookie.
  - [ ] **Refresh:** Attempt to refresh `studentPb.collection('students').authRefresh()`. Handle errors by clearing store.
  - [ ] **Locals:** Expose `event.locals.studentPb` and `event.locals.student`.
  - [ ] **Guards:**
    - Define `isCamperPath` check.
    - If `isCamperPath` and `!locals.student`, redirect to `/(camper)/login`.
    - If `!isCamperPath` (Staff App) and `locals.student` and `!locals.user` (Student trying to access staff area), redirect to `/(camper)`.
  - [ ] **Export:** Append `set-cookie` for `studentPb` to the response.

### Dev Notes

- **Existing Patterns:** Reuse the existing `CustomAuthStore` and `createPocketBase` helper functions found in `hooks.server.ts`. Do not rewrite them, just utilize them for the third instance.
- **Cookie Name:** Strictly use `pb_student_auth` as defined in the Architecture.
- **Collection Name:** The students collection is named `students`.
- **Public Routes:** Ensure `/(camper)/login` and `/(camper)/oauth` (if added later) are excluded from the redirect logic to prevent infinite loops.
- **Reference:** See `TekCamp/frontend/src/hooks.server.ts` for the existing Admin/User logic to replicate.
