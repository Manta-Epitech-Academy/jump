# Story 2.2: Sequential Markdown Viewer

**Status:** `Draft`
**Epic:** 2. The "Cockpit" & Subject Progression

### User Story

**As a** Student,
**I want** to read my subject instructions one step at a time and validate my progress,
**so that** I am not overwhelmed by a massive wall of text and stay focused.

### Acceptance Criteria

1.  **Subject Route:** Create `/(camper)/camper/[subjectId]/+page.svelte`.
2.  **Data Initialization:**
    - Server loads the `subjects` record and parses `content_structure` (JSON).
    - Server looks for a `steps_progress` record for this student/subject/event combination. If none exists, it creates one implicitly starting at the first step.
3.  **Sequential Rendering:**
    - The UI displays a sidebar (roadmap) of all steps (titles only).
    - The main view renders the Markdown for the _currently active step_ (`current_step_id`).
    - Future steps are locked and hidden. Past steps can be clicked and reviewed.
4.  **QCM Auto-Unlock:**
    - If a step has `validation.type === 'auto_qcm'`, render a form with radio buttons for the options at the bottom of the step.
    - Submitting the correct answer triggers a server action updating `steps_progress` and advancing `current_step_id`.
    - The UI reacts instantly with a success animation (e.g., green flash, advance to next step).
5.  **Manta Fallback:** After 3 consecutive wrong QCM attempts, the QCM locks and prompts the student to use the "Appeler un Manta" button (which will be built in Epic 3).

### Dev Notes

- Use a markdown parser like `marked` or `svelte-markdown`.
- The `content_structure` JSON schema is defined in the Architecture Doc. Use TypeScript interfaces to strongly type it.
- SvelteKit Form Actions will handle the QCM submissions. Use `sveltekit-superforms` if validation gets complex, or native progressive enhancement.
