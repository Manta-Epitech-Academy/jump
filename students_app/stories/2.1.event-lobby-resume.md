# Story 2.1: The Event Lobby & Progress Resume

**Status:** `Draft`
**Epic:** 2. The "Cockpit" & Subject Progression

### User Story

**As a** Student,
**I want** to see my assigned subject and my current overall progress when I log in,
**so that** I know exactly where to resume my work, even if it's day 3 of an internship.

### Acceptance Criteria

1.  **Dashboard Route:** The root `/(camper)/camper/+page.svelte` acts as the Lobby.
2.  **Event Fetching:** On load, the server fetches the student's `participations` record where the associated `event.date` falls on "today" (handling start-of-day/end-of-day bounds).
3.  **UI Data Display:**
    - Display the student's total XP and Level (Novice/Expert) proudly.
    - Display the `event.titre` and the associated `subjects` from the participation record.
4.  **"Start / Resume" Action:** If a subject is assigned, display a prominent "Démarrer l'atelier" or "Reprendre" button. This button routes the student to `/(camper)/camper/[subjectId]`.
5.  **Empty States:**
    - If no event is scheduled for today: Display a friendly "Aucun événement prévu aujourd'hui" message.
    - If an event is scheduled but no subject is assigned yet: Display "Ton sujet arrive... Le Manta prépare ta mission."
6.  **"Friendly Tech" Aesthetic:** The page uses the light-mode UI, `rounded-2xl` containers, Epitech blue/teal accents, and friendly copy.

### Dev Notes

- Use `now('Europe/Paris')` (or equivalent) in `+page.server.ts` to bound the event query to today.
- The `participations` relation should expand `event` and `subjects`.
- Ensure the design is strictly "desktop-first" but responsive.
