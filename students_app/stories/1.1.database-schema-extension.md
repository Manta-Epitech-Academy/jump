# Story 1.1: Database Schema Extension for Students

**Status:** `Approved`
**Epic:** 1. Foundation & Camper Authentication

### User Story
**As a** System,
**I want** to extend the PocketBase schema to support student authentication and session tracking,
**so that** students can log in securely via OTP and their progress is tracked persistently.

### Acceptance Criteria
1.  **`students` Collection Update:** The existing `students` collection is converted to an **Auth Collection** (or configured) to allow **Email/OTP** authentication. Password auth must be disabled. New fields (`xp`, `level`, `badges`) are added.
2.  **`subjects` Collection Update:** A new JSON field `content_structure` is added to store the ordered steps and QCM data.
3.  **`steps_progress` Collection Created:** A new collection is created to link `student`, `subject`, and `current_step_id`.
4.  **`portfolio_items` Collection Created:** A new collection is created for file uploads.
5.  **API Rules (RLS):** Basic API rules are applied to these collections to ensure students can only view/edit their own data (as defined in Architecture Section 6).
6.  **Type Generation:** The `pocketbase-types.ts` file is regenerated to reflect these changes (if using the automated type gen script).

### Tasks / Subtasks
*   [ ] **1. Backup Existing Schema**
    *   [ ] Export the current `pb_schema.json` (snapshot) before making changes, just in case.
*   [ ] **2. Modify `students` Collection**
    *   [ ] In PB Admin > Collections > `students`: Click "Settings" (gear icon).
    *   [ ] **Action:** Verify/Convert to "Auth Collection" (if it was Base).
    *   [ ] **Auth Options:** Enable `Email/Password` provider (required for OTP flow usually, even if we hide password UI) or specifically `OTP` if on PB v0.23+. **Disable** `Username/Password` login if possible/desired to force Email.
    *   [ ] **Add Fields:**
        *   `xp` (Number, default 0)
        *   `level` (Select: 'Novice', 'Apprentice', 'Expert')
        *   `badges` (JSON, default `[]`)
*   [ ] **3. Update `subjects` Collection**
    *   [ ] Add field: `content_structure` (JSON).
*   [ ] **4. Create `steps_progress` Collection**
    *   [ ] Create new Base collection.
    *   [ ] Add fields:
        *   `student` (Relation -> `students`, Cascade Delete: False)
        *   `event` (Relation -> `events`)
        *   `subject` (Relation -> `subjects`)
        *   `current_step_id` (Text/String - store the UUID from JSON)
        *   `unlocked_step_id` (Text/String)
        *   `status` (Select: `active`, `needs_help`, `completed`)
        *   `last_updated` (Date)
    *   [ ] **API Rules:**
        *   List/View: `student = @request.auth.id`
        *   Create/Update: `student = @request.auth.id`
*   [ ] **5. Create `portfolio_items` Collection**
    *   [ ] Create new Base collection.
    *   [ ] Add fields:
        *   `student` (Relation -> `students`)
        *   `event` (Relation -> `events`)
        *   `file` (File, limit: 1, mime: images)
        *   `url` (Url)
        *   `caption` (Text)
    *   [ ] **API Rules:**
        *   List/View: `student = @request.auth.id || @request.auth.collectionName = 'users'` (Staff access)
        *   Create: `student = @request.auth.id`
*   [ ] **6. Update Types**
    *   [ ] Run the type generator command (e.g., `npx pocketbase-typegen`) to update `src/lib/pocketbase-types.ts`.

### Dev Notes
